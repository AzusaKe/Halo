## 顶层结构

一个光环定义 JSON 文件包含以下顶层字段：

```json
{
  "id": "halo:example",
  "orientation_mode": "locked",
  "allow_angular_momentum": true,
  "sync_offset": [0.0, 0.0, 0.0],
  "layers": [ ... ],
  "animation": { ... },
  "hide_on_sleep": false,
  "positioning": { ... },
  "damping": { ... }
}
```

### `id`

- **类型**：字符串
- **必需**：是
- **格式**：`命名空间:名称`，例如 `"halo:yourhalo"`
- **说明**：光环的唯一标识符。命名空间映射到 `assets/` 下的文件夹——`halo:yourhalo` 表示定义文件位于 `assets/halo/` 下。你可以使用自己的命名空间（如 `mypack:myhalo`），此时定义文件应放在 `assets/mypack/` 下。后续通过 `/halo show <实体> <名称>` 引用时，模组会自动补全 `halo:` 前缀。

### `orientation_mode`

- **类型**：字符串枚举
- **必需**：否
- **可选值**：`"locked"`（默认）、`"free"`、`"sync"`
- **说明**：控制光环如何相对于实体头部旋转。详见 [朝向模式](#朝向模式-orientation-mode)。

### `allow_angular_momentum`

- **类型**：布尔值
- **必需**：否
- **默认值**：`false`
- **说明**：当设为 `true` 时，为光环的朝向添加角动量惯性效果。启用后，`locked` 和 `free` 模式下计算出的朝向不再瞬间到达，而是作为目标姿态被阻尼追赶——玩家快速转头时，光环会产生物理感的"滞后"。仅对 `locked` 和 `free` 模式有效，`sync` 模式下此字段无作用。角动量的阻尼速度和钳制角度由 `damping.angularMomentumFactor` 和 `damping.maxAngularMomentumDegrees` 控制。

### `sync_offset`

- **类型**：`[yaw, pitch, roll]` 三元组（度）
- **必需**：否
- **默认值**：`[0, 0, 0]`
- **说明**：仅在 `orientation_mode` 为 `"sync"` 时有效。定义光环与实体头部之间的固定旋转偏移量。欧拉角采用 YXZ 顺序。

### `layers`

- **类型**：图层对象数组
- **必需**：是（或提供旧版 `shape`）
- **说明**：定义光环的视觉组成。每个图层是一个独立的渲染单元。详见 [图层](#图层-layer)。

### `animation`

- **类型**：动画对象
- **必需**：否
- **说明**：光环级别的整体动画。结构体与图层级动画完全相同。当设置为 `{}` 时表示无动画。详见 [动画系统](#动画系统)。

### `hide_on_sleep`

- **类型**：布尔值
- **必需**：否
- **默认值**：`false`
- **说明**：当设为 `true` 时，实体睡觉（玩家在床上、狐狸休息等）期间光环会暂时停止渲染。实体醒来后自动恢复显示。设为 `false` 时始终渲染，不受睡眠状态影响。

### `positioning`

- **类型**：对象
- **必需**：是
- **说明**：光环整体的位置偏移和缩放。详见 [定位](#定位-positioning)。

### `damping`

- **类型**：对象
- **必需**：是
- **说明**：物理跟随参数。详见 [阻尼](#阻尼-damping)。

## 图层 (Layer)

每个图层定义如下：

```json
{
  "id": "optional_layer_name",
  "position": [0.0, 0.0, 0.0],
  "rotation": [0.0, 0.0, 0.0],
  "scale": 1.0,
  "glowing": true,
  "animation": { ... },
  "primitive": { ... }
}
```

### `id`

- **类型**：字符串
- **必需**：否
- **说明**：图层的可选名称，预留用于未来的动画编组功能。当前暂无实际作用。

### `position`

- **类型**：`[x, y, z]` 三元组（格）
- **必需**：否
- **默认值**：`[0, 0, 0]`
- **说明**：该图层在光环局部空间中的位置偏移。`[0, 0, 0]` 代表光环**原点**——该原点与阻尼物理计算出的跟随位置对齐。图层在空间中的实际位置 = 阻尼跟随位置 + 此处定义的偏移。

  在设计多层光环时，通过调整各图层的 Y 值可以创造出空间层次感——Y 值越大的图层在视觉上越靠"上"。

### `rotation`

- **类型**：`[pitch, yaw, roll]` 三元组（度）
- **必需**：否
- **默认值**：`[0, 0, 0]`
- **说明**：该图层的初始旋转角度，采用欧拉角 YXZ 顺序。

### `scale`

- **类型**：浮点数
- **必需**：否
- **默认值**：`1.0`
- **说明**：该图层的统一缩放倍率。`1.0` 为原始大小。此缩放与 `positioning.scale` 叠乘——最终大小 = 图层 `scale` × 全局 `positioning.scale`。

### `glowing`

- **类型**：布尔值
- **必需**：否
- **默认值**：`true`（当字段不存在时）
- **说明**：控制该图层是否以全亮度渲染（不受环境光照影响）。设为 `false` 时，图层会受到游戏内光照的影响（白天亮、夜晚暗）。

### `animation`

- **类型**：动画对象
- **必需**：否
- **说明**：该图层的独立动画。结构体与顶层动画完全相同。设为 `{}` 表示此图层无动画。详见 [动画系统](#动画系统)。

### `primitive`

- **类型**：对象
- **必需**：是
- **说明**：该图层的渲染图元。详见 [图元](#图元-primitive-与发光层-glow)。

## 图元 (Primitive) 与发光层 (Glow)

当前支持两种图元类型：

- **`billboard`**：无厚度的四边形面片
- **`ring`**：圆柱形圆环，具有半径和柱面宽度，内外表面可分别贴图

未来将支持更多图元类型：

- **`mesh`**：三维网格模型

### Billboard 图元

```json
{
  "type": "billboard",
  "texture": "halo:textures/halo/example.png",
  "size": [0.5, 0.5],
  "glow": { ... }
}
```

| 字段      | 类型             | 必需   | 说明                                                                                                                                                                                                                                                                                                                                        |
| --------- | ---------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `type`    | 字符串           | **是** | 固定值 `"billboard"`                                                                                                                                                                                                                                                                                                                        |
| `texture` | 字符串           | **是** | 贴图资源路径。格式：`命名空间:textures/halo/文件名.png`。路径以 `assets/` 为根目录，其中**命名空间**对应 `assets/` 下的文件夹名。例如 `halo:textures/halo/example.png` 中 `halo` 是命名空间，对应 `assets/halo/` 文件夹。使用自己的命名空间（如 `mypack`）时，将文件放在 `assets/mypack/` 下，路径写为 `mypack:textures/halo/example.png`。 |
| `size`    | `[width, depth]` | **是** | 面片在 XZ 平面上的尺寸 `[宽, 深]`（格）。通常设为正方形，如 `[0.5, 0.5]`。                                                                                                                                                                                                                                                                  |
| `glow`    | 对象             | 否     | 可选的发光叠加层。不存在时表示无发光效果。**当前版本此字段暂不可用，等待后续修复。** |

### Ring 图元

```json
{
  "type": "ring",
  "texture": "halo:textures/halo/example_outer.png",
  "inner_texture": "halo:textures/halo/example_inner.png",
  "size": [0.35, 0.08],
  "segments": 32
}
```

| 字段             | 类型             | 必需   | 说明                                                                                                                                                                                                                                                                            |
| ---------------- | ---------------- | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `type`           | 字符串           | **是** | 固定值 `"ring"`                                                                                                                                                                                                                                                                 |
| `texture`        | 字符串           | **是** | 外表面贴图资源路径。也可写作 `outer_texture`。                                                                                                                                                                                                                                  |
| `inner_texture`  | 字符串           | 否     | 内表面贴图资源路径。省略时内外表面共用 `texture`。当仅提供一张贴图时，内外表面均可见；当分别提供时，每张贴图只从其对应的一侧可见（面剔除）。                                                                                                                                    |
| `size`           | `[radius, width]`| **是** | `[半径, 柱面宽度]`（格）。`radius` 为圆环半径，`width` 为圆柱轴向宽度（Y 方向高度）。                                                                                                                                                                                          |
| `segments`       | 整数             | 否     | 多边形分段数，默认 `32`。值越大圆环越平滑，但顶点数也随之增加。                                                                                                                                                                                                                |

**旋转约定**：当 `rotation` 为 `[0, 0, 0]` 时，圆环平躺在 XZ 平面上（对称轴沿 -Y 方向），与 billboard 的默认朝向一致。纹理接缝位于 +X 轴方向。

**纹理映射**：U 沿圆周方向（0→1 = 一圈，接缝在 +X），V 沿柱面宽度方向（0 = 顶面 `+width/2`，1 = 底面 `-width/2`）。贴图在 U 方向首尾相接形成闭环。

### 发光层 (Glow)

> **⚠️ 注意**：发光层当前存在已知 bug，尚待修复。

发光层以加性混合（additive blending）渲染在基础贴图之上，产生自发光效果：

```json
{
  "texture": "halo:textures/halo/example_glow.png",
  "size": [0.5, 0.5],
  "color": 16766720,
  "alpha": 0.8
}
```

| 字段      | 类型             | 必需   | 说明                                                                      |
| --------- | ---------------- | ------ | ------------------------------------------------------------------------- |
| `texture` | 字符串           | **是** | 发光贴图资源路径。                                                        |
| `size`    | `[width, depth]` | **是** | 发光面片尺寸（格）。通常与基础贴图相同。                                  |
| `color`   | 整数             | **是** | 发光颜色，以 0xRRGGBB 格式的整数表示。例如 `16766720`（0xFFCC00，金色）。 |
| `alpha`   | 浮点数           | **是** | 基础透明度，范围 0.0–1.0。                                                |

## 动画系统

动画系统使用数学函数来描述位置偏移和旋转角度随时间的变化。动画可以定义在顶层（`animation`，影响整个光环）或图层级（`layers[i].animation`，仅影响该图层）。

> **傅里叶级数的力量**：由于同一轴上的多个动画项会求和叠加，而 `sin` 和 `cos` 构成了傅里叶级数的基础，理论上你可以通过叠加不同频率、振幅和相位的正弦/余弦项来逼近**任何周期运动**——从简单的上下浮动到复杂的轨迹，甚至一个微缩小人在跳舞。

### 结构

```json
{
  "offset": {
    "x": [ ... ],
    "y": [ ... ],
    "z": [ ... ]
  },
  "rotation": {
    "yaw": [ ... ],
    "pitch": [ ... ],
    "roll": [ ... ]
  }
}
```

- **`offset`**：位置偏移动画，按轴分为 `x`、`y`、`z` 三个可选数组
- **`rotation`**：旋转动画，按轴分为 `yaw`（偏航）、`pitch`（俯仰）、`roll`（翻滚）三个可选数组

每个轴的值是一个**动画项数组**。同一轴上的多个动画项会**求和叠加**，因此可以组合多个函数来产生复杂运动。整个动画块、每个分组、每条轴都是可选的——不需要的部分可以省略。

### 单位

| 动画类型   | 轴                     | 单位         |
| ---------- | ---------------------- | ------------ |
| `offset`   | `x`, `y`, `z`          | **格**（米） |
| `rotation` | `yaw`, `pitch`, `roll` | **度**       |

### 动画函数

共有三种动画函数可用：

#### `sin` — 正弦振荡

```json
{ "function": "sin", "A": 0.01, "omega": 0.5, "phi": 0.0 }
```

| 参数    | 必需   | 默认值 | 说明                                                           |
| ------- | ------ | ------ | -------------------------------------------------------------- |
| `A`     | **是** | —      | 振幅。                                                         |
| `omega` | **是** | —      | 角频率（单位：π）。`omega = 1` 对应周期 2 秒。值越大振荡越快。 |
| `phi`   | 否     | `0.0`  | 相位偏移（弧度）。用于让多个图层的振荡错开。                   |

**公式**：`value(t) = A · sin(omega · π · t + phi)`

#### `cos` — 余弦振荡

```json
{ "function": "cos", "A": 0.01, "omega": 0.5, "phi": 0.0 }
```

参数和公式与 `sin` 相同，仅将 `sin` 替换为 `cos`。`cos` 在 `t=0` 时从振幅峰值开始，`sin` 从零开始。

**公式**：`value(t) = A · cos(omega · π · t + phi)`

#### `linear` — 线性变化

```json
{ "function": "linear", "start": 0.0, "speed": 45.0 }
```

| 参数    | 必需   | 默认值 | 说明                                              |
| ------- | ------ | ------ | ------------------------------------------------- |
| `start` | 否     | `0.0`  | 初始值（`t=0` 时的值）。                          |
| `speed` | **是** | —      | 变化速率（单位/秒）。正值正向变化，负值反向变化。 |

**公式**：`value(t) = start + speed · t`

**典型用法**：

- 持续旋转：`{ "function": "linear", "speed": 45 }` → 每秒顺时针旋转 45°
- 缓慢反转：`{ "function": "linear", "speed": -6 }` → 每秒逆时针旋转 6°

## 朝向模式 (Orientation Mode)

朝向模式控制光环在空间中如何相对于实体旋转。理解朝向模式需要先区分两组概念：

**光环定义中的概念工具**（仅在光环定义的相对空间中起作用）：

- **光环原点**：图层 `position` 偏移的参考中心。每个图层的 `position` 是相对这个原点定义的。
- **光环法线**：默认方向为竖直向下（-Y）。用于确定光环的"正面"朝向。

**计算得出的最终姿态**（由阻尼物理和朝向模式共同决定）：

- **光环位置**：光环最终在游戏世界中的空间坐标。
- **光环朝向**：光环最终在游戏世界中的三维旋转方向。

**渲染时的对齐过程**：先得出光环位置和光环朝向，然后将光环原点移动到光环位置，再旋转整个光环直到光环法线与光环朝向同向平行——这样就得到了最终需要渲染的物体。原点/法线与位置/朝向是两组不同层面的概念，不能混为一谈。

### `locked`（默认）

- 光环朝向**始终指向玩家头部中心**
- 通过玩家头部的**球面极点**（上下两个极点）来确定光环绕法线的旋转——光环像指南针一样稳定指向这两个极点，不会绕法线随意旋转
- 当 `allow_angular_momentum` 为 `true` 时，指向头部中心的朝向变为目标，实际朝向带有角动量惯性地平滑追赶
- 适合需要稳定朝向的光环，如默认的 ring_default

### `free`

- 光环朝向**同样始终指向玩家头部中心**
- 但光环**绕法线的旋转不受控制**——可以在法线周围自由旋转
- 当 `allow_angular_momentum` 为 `true` 时，同样会为朝向添加角动量惯性
- 适合不关心具体旋转角度的光环

### `sync`

- **初始姿态**由 `locked` 模式的首帧确定
- 之后光环的旋转**完全跟随实体头部**——玩家转头时光环也同步旋转
- 可以通过 `sync_offset` 设置与头部之间的固定旋转偏移
- `allow_angular_momentum` 对此模式无效
- 适合需要与实体保持固定相对角度的光环（灵感来源于 Blue Archive）

### 模式对比

| 特性               | `locked`       | `free`   | `sync`            |
| ------------------ | -------------- | -------- | ----------------- |
| 指向头部中心       | ✓              | ✓        | ✗（跟随头部旋转） |
| 绕法线旋转         | 由球面极点稳定 | 不受控制 | 跟随头部          |
| 角动量惯性         | ✓              | ✓        | ✗                 |
| 初始姿态来源       | —              | —        | `locked` 的首帧   |
| `sync_offset` 生效 | ✗              | ✗        | ✓                 |

## 定位 (Positioning)

```json
{
  "offset": [0.0, 0.4, 0.35],
  "scale": 1.5
}
```

| 字段     | 类型        | 必需   | 默认值 | 说明                                                                     |
| -------- | ----------- | ------ | ------ | ------------------------------------------------------------------------ |
| `offset` | `[x, y, z]` | **是** | —      | 光环整体相对于实体头部锚点的位置偏移（格）。`y` 正值向上，`z` 正值向后。 |
| `scale`  | 浮点数      | 否     | `1.0`  | 光环整体的缩放倍率。与各图层的 `scale` 叠乘。                            |

**典型值**：`offset: [0, 0.4, 0.35]` 将光环放置在头部上方 0.4 格、后方 0.35 格处——这是类人物体的推荐默认值。

## 阻尼 (Damping)

阻尼控制光环如何平滑地跟随实体运动。模组采用帧率无关的指数衰减算法。

```json
{
  "linearFactor": 0.5,
  "angularFactor": 0.1,
  "maxLinearDistance": 0.5,
  "maxAngularDegrees": 180.0,
  "angularMomentumFactor": 0.3,
  "maxAngularMomentumDegrees": 45.0
}
```

| 字段                        | 类型   | 必需   | 默认值 | 说明                                                                                                                                                   |
| --------------------------- | ------ | ------ | ------ | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `linearFactor`              | 浮点数 | **是** | —      | 线性跟随速度。`0` = 不跟随（光环停留在原地），`1` = 瞬间跟随（光环完全贴紧实体）。典型范围 0.3–0.8。                                                   |
| `angularFactor`             | 浮点数 | **是** | —      | 角度跟随速度。`0` = 不跟随旋转，`1` = 瞬间跟随旋转。通常设为比 `linearFactor` 更低的值（如 0.1），让旋转更柔和。                                       |
| `maxLinearDistance`         | 浮点数 | **是** | —      | 线性钳制距离（格）。光环与实体的距离达到此值后将被**钳制**在此距离，不会继续拉远。当速度放缓后，光环会自然回弹到正常跟随位置。                         |
| `maxAngularDegrees`         | 浮点数 | **是** | —      | 角度钳制值（度）。光环与实体的角度差达到此值后将被**钳制**在此角度，不会继续偏转。当速度放缓后自然回弹。                                               |
| `angularMomentumFactor`     | 浮点数 | 否     | `0.3`  | 角动量阻尼系数。仅在 `allow_angular_momentum` 为 `true` 时生效。`0` = 光环朝向冻结（不追赶目标），`1` = 瞬间到达目标朝向（无惯性）。典型范围 0.2–0.5。 |
| `maxAngularMomentumDegrees` | 浮点数 | 否     | `45.0` | 角动量最大偏差角度（度）。光环的实际朝向与目标朝向之间的角度差超过此值后将被**钳制**，防止光环在快速转向时偏离过大。                                   |

> **钳制不是传送**：钳制意味着光环被"卡"在最大距离/角度处，而非直接跳到实体旁边。传送（如跨维度）有单独的检测机制，会使光环瞬间归位。钳制用于防止正常游戏中因快速移动造成的光环脱离过远。

**推荐配置**：

| 风格     | `linearFactor` | `angularFactor` | `maxLinearDistance` | 效果                            |
| -------- | -------------- | --------------- | ------------------- | ------------------------------- |
| 轻盈飘浮 | 0.3–0.4        | 0.05–0.1        | 0.5–1.0             | 光环有明显滞后感，飘逸          |
| 平衡     | 0.5–0.6        | 0.1–0.15        | 0.5                 | 适中的跟随感                    |
| 紧致跟手 | 0.8–1.0        | 0.2             | 0.3–0.5             | 光环几乎紧贴实体，适合 HUD 风格 |

## 常见模式与技巧

### 1. 轻微上下浮动

为每个图层添加 Y 轴正弦振荡，是最基础也是最常用的动画。让光环看起来"活着"：

```json
"animation": {
  "offset": {
    "y": [{ "function": "sin", "A": 0.01, "omega": 0.5 }]
  }
}
```

### 2. 多层光环 + 相位错开

多个图层叠加，每层使用不同的 `phi` 让浮动节奏错开，产生更有机的复合运动：

```json
"layers": [
  {
    "position": [0, 0.000, 0],
    "animation": {
      "offset": {
        "y": [{ "function": "sin", "A": 0.01, "omega": 0.5 }]
      }
    },
    ...
  },
  {
    "position": [0, 0.015, 0],
    "animation": {
      "offset": {
        "y": [{ "function": "sin", "A": 0.01, "omega": 0.5, "phi": 0.5 }]
      }
    },
    ...
  },
  {
    "position": [0, 0.045, 0],
    "animation": {
      "offset": {
        "y": [{ "function": "sin", "A": 0.01, "omega": 0.5, "phi": 1.0 }]
      }
    },
    ...
  }
]
```

> 各层 Y 的 `position` 逐渐增大（0 → 0.015 → 0.045），形成空间层次。`phi` 逐层递增让各层的浮动不同步。

### 3. 持续旋转

使用 `linear` 函数让图层绕法线持续旋转。不同图层不同速度不同方向，产生复杂的视觉效果：

```json
"animation": {
  "rotation": {
    "yaw": [{ "function": "linear", "speed": -0.1 }]
  }
}
```

> `speed` 为负值表示逆时针旋转。组合多个图层各自以不同速度旋转可制作多层旋转光环。

### 4. HUD 风格光环

利用图层的 `position` 偏移将元素放置在光环平面的不同位置（而非都放在原点），并配合不同动画，可制作类似科幻 HUD 的效果：

```json
{
  "position": [0.22, 0.0, 0.205],
  "animation": {
    "rotation": {
      "yaw": [{ "function": "linear", "speed": 45 }]
    }
  },
  ...
}
```

> 将元素偏移到 `[0.22, 0, 0.205]` 位置（光环平面的角落区域），再让它高速旋转。

### 5. 纹理方向

回顾纹理方向规则：贴图的上下在游戏中永远是上下（不翻转），且从玩家头部向外看时贴图为正向，从外部看时左右翻转。对称图案不受影响。

## 附录：旧版格式

旧版定义使用 `shape` 字段代替 `layers`，模组在加载时自动将其转换为图层格式。支持两种 `shape.type`：

### `billboard`

```json
{
  "id": "halo:example",
  "shape": {
    "type": "billboard",
    "texture": "halo:textures/halo/example.png",
    "size": [0.5, 0.5]
  },
  "animation": { ... },
  "positioning": { ... },
  "damping": { ... }
}
```

等价于具有一个图层的 `layers` 格式。

### `multi_billboard`

```json
{
  "id": "halo:example",
  "shape": {
    "type": "multi_billboard",
    "layers": [
      { "texture": "...", "size": [0.5, 0.5], "scale": 1.0 },
      { "texture": "...", "size": [0.5, 0.5], "scale": 1.0 }
    ]
  },
  ...
}
```

> **注意**：`multi_billboard` 没有位置概念——它只是简单地将所有图层叠加在一起。与新 `layers` 格式不同，它不支持每层的 `position` 偏移、独立动画、发光等高级功能。

> **建议**：新编写的光环定义应使用 `layers` 格式，功能更丰富（支持每层独立动画、发光、旋转等）。旧版格式仅用于兼容已有的定义文件。
